Privacy Policy — FocusFlow

Last updated: December 21, 2025

Overview

FocusFlow is a productivity app that helps users stay focused by offering timers, task management, gamification, and an Android-native app-blocking service. We prioritise privacy and keep data collection to the minimum required for app functionality. This page explains what the app does, what data (if any) is collected, how it's used, and how you can control it.

What the app does

  • Provides Pomodoro and Deep Focus timers.
  • Allows users to create tasks, earn XP and badges, and track streaks.
  • Offers a native Android foreground service that can monitor which app is in the foreground and show a blocking overlay during focus sessions (to help prevent access to distracting apps the user chooses to block).
  • Supports optional cloud sync (Supabase) for cross-device data when the user signs up and explicitly enables sync.

App access & required permissions

To provide app-blocking and reminders, FocusFlow requests the following Android permissions on devices where those features are used:

  • Usage access (PACKAGE_USAGE_STATS) — to detect which app is currently in the foreground. This is used only to determine when a blocked app is opened so the native service can show an overlay or take blocking action.
  • Display over other apps (SYSTEM_ALERT_WINDOW) — to show the blocking overlay. If not granted, the app falls back to notifications.
  • Notifications — to deliver session reminders and blocking notifications.
  • Foreground service — to run the native blocking service reliably in the background.

Permissions are requested only when a feature that needs them is enabled. The user is explicitly prompted and must grant these permissions in system settings.

Data collected

We collect only the data necessary for the app's functions. No sensitive personal data or secret keys are embedded in this policy.

  • Account information (optional): If you create an account, we store the email and username to identify your account. This is required only for cloud sync and cross-device features.
  • App usage signals (local): The native Android service may read foreground app package names to enforce blocking rules. These package names are used locally on-device to decide when to show the blocking overlay. This data is not used for advertising and is not sent to third parties by default.
  • Local settings & progress: Preferences, blocked-app lists, timer history, points, badges and streaks are stored locally (in an encrypted Hive database where sensitive data is encrypted).
  • Optional cloud sync: If you enable cloud sync, the app will upload selected sync items (account id, username, points, badges, blocked app list, and basic session stats) to the project's Supabase backend. Sync is optional and can be disabled from settings; you can delete your account and synced data by contacting support.

How we use data

  • To operate the core functionality (blocking selected apps, timers, and task management).
  • To provide optional cross-device sync and to restore user state when the user signs in.
  • To send service notifications and reminders.

Data sharing & third parties

We do not share personal data with advertisers or analytics companies. We do not include third-party advertising SDKs in the app. Data stored in the optional cloud sync may be stored in Supabase (a managed backend); operations on that backend are governed by Supabase's policies. We never embed secrets or private keys in the app repository.

Ads

FocusFlow does not display third-party advertisements and does not include ad SDKs.

Content rating & target audience

Content rating: Suitable for Everyone (no explicit content).

Target audience: general users aged 13 and above. The app is not intended for children under 13; if your local law requires parental consent for younger users, please comply when providing access.

Health, financial, and government features

FocusFlow is a productivity tool and does not provide medical, financial, or government-specific services. It makes no clinical claims or treatment recommendations and is not a medical device. It does not require government approvals.

Data safety summary for Google Play

  • Data types collected: Account info (email, username) — user-provided when creating an account; Device or other identifiers — foreground app package names (used locally for blocking); App content — blocked-app lists and timer/task content (local user data); Diagnostics — none collected for analytics by default.
  • Collected for: App functionality, optional cloud sync, user support and restoration.
  • Shared with third parties: Only optional synced data to Supabase if the user opts in. No advertising or analytics sharing.

User controls & data rights

  • Opt-out of cloud sync at any time in Settings; disabling sync prevents further uploads.
  • Delete local app data via the app settings or by uninstalling the app; encrypted local data is removed when the app's storage is cleared.
  • To delete cloud-stored data or request account removal, contact support (see below).
